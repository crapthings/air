# 内容审查台设计

日期：2026-03-07

## 目标

新增一个面向中文自媒体用户的“内容审查台”页面，统一覆盖三类场景：

- 发布前自检
- 内容复核
- 申诉辅助

该页面与现有信息评分器并列存在，但评价目标不同。信息评分器回答“值不值得看”，内容审查台回答“能不能发、为什么危险、怎么改、能否申诉”。

## 页面命名

页面正式名称：内容审查台

子模式：

- 发布前自检
- 内容复核
- 申诉辅助

推荐交互：顶部用分段切换控制三种模式，不拆成三个独立页面。

## 核心输入

输入字段：

- 标题
- 正文
- 封面文案 / 口播摘录
- 平台（通用 / 小红书 / 抖音 / 视频号 / 公众号）

设计原则：

- 标题和正文分开，因为中文平台常见风险首先来自标题党、极限词、封面诱导
- 封面文案/口播摘录单独存在，因为视频型内容风险经常在口播和封面层面显现
- 平台先做轻量选择，不引入复杂规则引擎，只作为提示词上下文

## 输出结构

页面输出分四块信息：

### 1. 总体风险

展示：

- 总风险等级
- 审查结论
- 是否建议发布

建议文案：

- 可发布
- 建议修改后发布
- 高风险，不建议直接发布

### 2. 风险指标

建议首版做 7 项：

- 违规风险
- 广告营销风险
- 医疗健康风险
- 金融收益风险
- 低俗攻击风险
- 引流导流风险
- 误导标题风险

这些指标都按 0-100 打分。分数越高表示风险越高。

### 3. 命中点

结构化返回：

- 命中内容
- 风险原因
- 严重程度
- 平台关注点

目标不是泛泛总结，而是告诉用户“哪一句危险、为什么危险”。

### 4. 整改与申诉

随模式变化：

- 发布前自检：输出修改建议
- 内容复核：输出复核判断依据
- 申诉辅助：输出是否建议申诉、可申诉点、申诉理由方向

## 中文自媒体重点检测点

首版重点覆盖：

- 极限词与绝对化表达
- 虚假承诺与结果保证
- 医疗、健康、减肥、保健暗示
- 金融收益承诺、荐股、带单
- 引流导流、私联、加微、进群
- 低俗、擦边、侮辱、攻击
- 标题党、封面党、过度诱导点击
- 暗示规避平台规则
- 假借专家、机构、官方背书
- 敏感身份、群体、社会议题风险

## 页面布局

推荐沿用现有产品的全宽三栏节奏：

- 左栏：模式切换 + 输入区
- 中栏：风险指标卡 + 命中点列表
- 右栏：总体风险 + 整改与申诉 + 审查剖面

其中“审查剖面”可以类比现有“分析剖面”，但内容换成合规视角。

## 审查剖面建议字段

- 核心问题
- 风险风格
- 平台敏感点
- 建议动作

## 与现有系统关系

现有页面：信息评分器

新增页面：内容审查台

二者共享：

- OpenRouter 配置
- 默认提示词 fallback 机制
- 三栏布局语言

二者独立：

- 指标体系
- JSON schema
- 右侧剖面字段

## 实现建议

优先级：

1. 页面壳和模式切换
2. 输入结构与输出卡片
3. 新的审查 prompt 和 schema
4. 命中点列表和整改建议
5. 申诉辅助视图差异

## 风险与边界

- 首版不要声称“平台官方判定结果”，只能表达“风险评估”
- 不要承诺准确规避平台审核
- 不做法律意见
- 不做医疗或金融真实性背书
- 平台差异先做轻量提示，不做重规则引擎
